Markdown files with project write‑ups.
Principles:
Frame each project as a story: challenge → approach → outcome.
Emphasize measurable impact (time saved, costs reduced, accuracy improved).
Highlight ethical and inclusive data use (GDPR, accessibility, stewardship).
Balance analyst and engineer perspectives: insights for decision‑makers, pipelines for efficiency.
Use clear visuals and analogies to make complex systems accessible to non‑technical audiences.
